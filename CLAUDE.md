## Commands

- `npm run dev` — start Vite dev server (expects backend at `VITE_API_URL`, default `http://localhost:3001`)
- `npm run build` — type-check (`tsc -b`) then build with Vite
- `npm run lint` — ESLint (flat config in `eslint.config.js`)
- `npm run preview` — preview the production build

There is no test runner configured.

## Architecture

React 19 + TypeScript + Vite SPA for a task CRUD app. The backend is external (any REST server at `VITE_API_URL` exposing `/tasks`).

**Routing — TanStack Router, file-based.** Routes live in `src/routes/` (`__root.tsx`, `index.tsx`, `task.$id.tsx`). `src/routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin/vite` (configured in `vite.config.ts`) — never hand-edit it. The router is wired up in `src/main.tsx` and wrapped in `QueryClientProvider`.

**Data layer — TanStack Query + thin fetch wrapper.**
- `src/apiClient.ts` is a generic `apiClient<T>(input, init)` wrapping `fetch`, prefixing `VITE_API_URL`, setting JSON headers, and throwing on non-ok responses.
- Queries and mutations are colocated with their feature in a `useXxx` hook (see `src/features/tasks/useTasks.ts`). The convention is to return both data and mutation handlers from a single hook so routes/components stay declarative.

**Feature-based structure** (from `.github/instructions/architecture.md`):
- `src/components/` — shared, generic UI primitives (`Button`, `Input`, `Textarea`)
- `src/features/<feature>/` — feature-scoped components, hooks, and logic (currently only `tasks/`)
- `src/routes/` — route components; they should delegate to feature modules rather than holding business logic
- `src/hooks/` — cross-cutting hooks (not feature-specific)
- `src/type.d.ts` — shared domain types (e.g. `Task`)

## Project conventions

**React Query cache updates — prefer `setQueryData` over `invalidateQueries`** (from `.github/instructions/react-query.md`). After a mutation succeeds, update the cache directly rather than triggering a refetch. Example in `src/features/tasks/useTasks.ts` — the add-task mutation uses `queryClient.setQueryData<Task[]>(["tasks"], old => [...(old ?? []), data])`. The delete/update mutations in that same file currently still use `invalidateQueries` and should be migrated to `setQueryData` when touched.

**Styling — Tailwind CSS v4** via `@tailwindcss/vite` (no `tailwind.config.js`; configured through the Vite plugin). `src/index.css` just contains `@import "tailwindcss"`.
